# task_1 피드백: 서버 생성 흐름 누락

## 받은 피드백

서버를 생성할 때 단순 생성 버튼이 아니라 게임을 고르고, 포트를 설정하고, SSH로 연결한 서버에서 열지 지금 실행 중인 PC에서 열지 선택해야 한다.

UI는 최대한 간결해야 한다. Docker 컨테이너가 생성됐는지 확인하는 방법과 SSH를 통해 생성한 서버 정보를 어떻게 가져올지도 정리해야 한다.

## 정정해야 할 기준

- 서버 생성은 `실행 위치 -> 게임 -> 포트/메모리/EULA -> 생성` 흐름이어야 한다.
- 실행 위치는 `지금 PC`, `SSH 서버`, `클라우드 서버`를 구분한다.
- SSH는 장기 제어 수단이라기보다 Agent 설치/연결을 위한 부트스트랩 수단으로 우선 본다.
- Docker 컨테이너 생성 여부는 Agent가 Docker label 기반으로 조회하는 방식이 안정적이다.
- UI는 마법사처럼 과하게 복잡하게 만들기보다 한 화면 설정 패널로 간결하게 제공한다.

## 반영 내용

- Desktop 서버 관리 화면에 `서버 생성 설정` 패널을 추가한다.
- 생성 요청에 실행 위치, SSH 정보, 게임 템플릿, 내부/외부 포트, 메모리, EULA 동의를 포함한다.
- Agent Docker CLI 생성 명령에 `remote-game-server.*` label을 붙인다.
- Docker 컨테이너 확인 기준을 문서화한다.
