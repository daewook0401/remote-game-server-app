# task_3 HAProxy 설치 흐름 피드백

## 받은 피드백

외부망 노트북 또는 점프 노드에 HAProxy가 설치되어 있지 않은 경우, 앱이 바로 실패하거나 자동으로 설치를 강행하면 안 된다.

원하는 흐름은 다음과 같다.

```text
1. HAProxy 설치 여부 감지
2. 설치되어 있지 않으면 우선 실패 상태로 사용자에게 알림
3. "HAProxy를 서버에 설치하시겠습니까?" 확인
4. sudo 비밀번호 입력
5. OS와 패키지 매니저 감지
6. HAProxy 설치 시도
7. 설치 후 다시 HAProxy 확인
```

## 반영 원칙

- HAProxy 자동 설치는 사용자 승인 후에만 수행한다.
- 설치에는 sudo 비밀번호를 별도 입력받는다.
- SSH key 인증을 쓰더라도 sudo 비밀번호가 필요할 수 있으므로 설치 모달에서 별도로 받는다.
- 지원 패키지 매니저는 `apt-get`, `dnf`, `yum`, `pacman`, `zypper`, `apk` 순서로 감지한다.
- 설치 실패는 감추지 않고 사용자에게 실패 메시지로 보여준다.

## 재발 방지 기준

인프라 패키지 설치, 방화벽 변경, 서비스 등록처럼 원격 서버 상태를 바꾸는 작업은 감지 후 즉시 실행하지 않고 사용자 확인과 비밀번호 입력 단계를 둔다.
