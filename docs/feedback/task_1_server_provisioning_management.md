# task_1 피드백: 서버 생성과 관리 흐름

## 받은 피드백

사용자는 로컬에서 프로그램을 실행해 자기 서버에서 Minecraft 서버를 구현하고 싶다.

Amazon 서버 같은 클라우드 서버에서 구현하는 경우에는 서버에 접속할 수 있는 방법을 제공해야 한다. 사용자가 클릭만으로 서버를 생성하거나 관리할 수 있어야 하며, Docker 콘솔에 접속하는 기능도 필요하다.

## 정정해야 할 기준

- 제품 범위는 단순히 게임 서버 포트를 외부에 공개하는 기능에 그치지 않는다.
- 설치형 프로그램은 서버 생성, 서버 연결, Docker 컨테이너 관리, 콘솔 접속까지 포함하는 관리 도구여야 한다.
- 로컬 서버와 클라우드 서버를 모두 대상 환경으로 본다.
- 클라우드 서버는 처음부터 완전 자동 생성이 어렵더라도, 접속 정보 등록과 Agent 설치/연결 흐름을 제공해야 한다.
- Docker 콘솔 접근은 사용자가 실제 서버를 운영하는 데 필요한 핵심 기능으로 본다.

## 반영할 제품 흐름

1. 사용자가 대상 환경을 선택한다.
   - 로컬 서버
   - 기존 원격 서버
   - 클라우드 서버
2. 원격 서버인 경우 접속 정보를 등록한다.
   - SSH host
   - SSH port
   - 사용자명
   - 인증 방식
3. 프로그램이 Agent 설치 또는 연결 상태를 확인한다.
4. Agent가 Docker 상태를 확인한다.
5. 사용자가 Minecraft 서버 템플릿을 선택한다.
6. 프로그램이 Docker 컨테이너를 생성/시작/중지/삭제한다.
7. 사용자가 컨테이너 로그와 콘솔에 접근한다.
8. 필요 시 Agent + FRP로 외부 공개를 시작한다.
