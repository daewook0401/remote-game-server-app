# task_2 Agent 포트 보안 피드백

## 피드백

- 테스트 서버에 18080 포트로 Agent를 설치했는데 보안 문제가 생길 수 있다.
- Agent는 Docker 컨테이너 생성, 삭제, 콘솔 명령 등 고위험 작업을 수행하므로 외부 공개 방식에 보안 조치가 필요하다.

## 판단 기준

- Agent API는 일반 게임 접속 포트처럼 공개하면 안 된다.
- 원격 제어 포트는 기본적으로 비공개 또는 제한 공개되어야 한다.
- 외부에서 접근해야 한다면 최소한 토큰 인증과 방화벽 허용 IP 제한이 필요하다.
- 더 안전한 기본 흐름은 SSH 터널을 통해 Desktop에서 `127.0.0.1:18080`처럼 접근하는 방식이다.

## 다음 작업 반영

- task_2 구현계획에 Agent 포트 보안 조치를 추가한다.
- 서버 상세 자동 상태 확인은 Agent URL이 보안상 위험한 구성을 사용할 수 있음을 사용자에게 알려야 한다.
- 장기적으로는 Agent 자동 설치 기본값을 `127.0.0.1:18080`로 두고 SSH 터널 연결을 우선하는 구조를 검토한다.
- 외부망 접근 서버와 내부망 대상 서버가 분리된 환경을 위해 점프 서버 경유 접속 옵션을 검토한다.
- 점프 서버 경유 접속은 Agent 설치, 상태 확인, 콘솔, 서버 삭제 정리 명령이 모두 같은 경로를 사용해야 한다.
