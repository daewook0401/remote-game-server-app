# task_3 분리 우선 작업 규칙 피드백

## 받은 피드백

`ServerManagementPage.tsx`처럼 큰 파일에 기능을 계속 추가하면 작업 시간이 길어지고 검수 범위가 커진다.

앞으로는 한 번에 묶어서 구현하지 말고, 파일이 커지거나 책임이 섞이는 작업은 먼저 분리 계획을 세운 뒤 진행한다.

## 반영 원칙

- 큰 TSX 파일에 새 기능을 바로 붙이지 않는다.
- UI 모달, 순수 모델/요청 조립, 원격 IPC 클라이언트, 페이지 상태 관리는 가능한 한 분리한다.
- 분리가 필요한 작업은 구현 전 계획에 분리 대상을 먼저 적는다.
- 이미 커진 파일은 기능 추가보다 분리를 우선 진행한다.

## 현재 적용

이번 작업은 이미 구현이 진행된 상태이므로 추가 승인 대기 없이 `ServerManagementPage.tsx` 분리를 계속 진행한다.
