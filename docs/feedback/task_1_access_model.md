# task_1 피드백: 접속 기능 실행 방식과 검증 범위

## 받은 피드백

1. 웹 UI가 아니라 프로그램 파일로 설치해서 실행하는 방식이다. 디자인도 실제 설치형 프로그램처럼 구성한다.
2. 현재 작업 공간에서는 수동 TCP 검증을 할 수 없다. 추후 Git 레포가 추가될 때 검증 절차를 넣을 수 있도록 준비한다.
3. UDP, Nginx Stream은 실제 접속 후 원격 수정이 어려운 부분이므로, 사용자에게 안내 가이드가 갈 수 있도록 한다.

## 정정해야 할 기준

- "웹 UI" 표현은 "설치형 프로그램 UI" 또는 "프로그램 화면"으로 정정한다.
- 현재 단계의 검증 계획에 "실제 TCP 접속 수동 검증"을 완료 조건처럼 넣지 않는다.
- TCP 접속 검증은 추후 Git 레포와 실행 환경이 추가된 뒤 재현 가능한 검증 절차로 편입한다.
- UDP와 Nginx Stream은 자동 원격 수정 대상이 아니라 사용자 안내 가이드 제공 대상으로 우선 분류한다.

- 구현계획서 작성 시 UDP, Nginx Stream은 자동화 범위와 안내 범위를 분리해서 적는다.
