# task_1 피드백: Go 검증과 UI 구조 분리

## 받은 피드백

Go를 추가 설치했으니 Go 검증을 다시 실행해야 한다. 안 되면 터미널 재실행, 직접 경로 확인 등으로 실행하지 못한 부분을 실행해야 한다.

개발 트리를 보니 `App.tsx`에 UI가 몰려 작업될 것처럼 보인다. 레이아웃, 컴포넌트, 페이지를 필요한 단위로 나누어 출력하도록 구조를 분리해야 한다. 라우터 설정을 넣어도 된다.

## 반영 기준

- PATH에서 `go`가 잡히지 않으면 설치 경로를 찾아 직접 실행한다.
- Go 단위 테스트가 없는 상태로 검증 성공이라고만 기록하지 않는다.
- Desktop UI는 `App.tsx`에 화면을 몰아넣지 않는다.
- 화면은 layout, page, component, data, type 단위로 분리한다.
