## 오늘 결정한 사항

- 서버 선택을 최상위 흐름으로 두고, 콘솔은 선택된 게임 서버 옆에서 접근하는 방향을 검토한다.
- 사이드바에서는 콘솔과 외부 공개 메뉴를 제거하고 서버 선택, 안내 가이드만 남기는 방향을 검토한다.
- 서버 카드의 상태 확인 버튼은 제거 대상으로 검토한다.
- 서버 상세 화면에 들어가면 Agent 상태 확인을 자동으로 한 번 실행해 자연스럽게 서버 상태와 컨테이너 목록을 불러오는 방향을 검토한다.
- Agent 18080 포트는 Docker 제어 API이므로 외부 공개와 토큰 누락 위험을 보안 이슈로 별도 확인한다.
- Agent 관련 보안 조치 방법을 승인 검토용 기술 문서로 정리한다.
- Agent token은 앱이 자동 생성/적용하고, 서버 삭제 시 원격 Agent, 데이터, 18080 방화벽 규칙 정리 옵션을 제공하는 방향을 검토한다.
- 외부망 접근 서버와 내부망 대상 서버가 분리된 환경을 위해 점프 서버 경유 접속 옵션을 후속 계획에 반영한다.
- 서버 등록 플로우는 SSH 확인, Agent 설치와 포트 설정, Agent 확인 완료 후 서버 등록 순서로 강제하는 방향을 후속 계획에 반영한다.
- 클릭 가능한 버튼은 커서와 시각적 반응으로 구분하고, 긴 요청에는 로딩 상태와 중복 클릭 방지를 추가하는 방향을 후속 계획에 반영한다.
- 점프 서버 경유 시 Agent 포트 설정은 점프 서버가 아니라 실제 Agent가 설치된 내부망 대상 서버에 적용하는 방향을 후속 계획에 반영한다.
- 등록된 서버 상세 화면에서는 `Agent 설치` 버튼을 제거하고 `Agent 업데이트`만 유지하는 방향을 후속 계획에 반영한다.

## 오늘 받은 피드백

- 서버가 없으면 콘솔과 외부 공개 기능도 독립적으로 의미가 약하므로 서버 선택을 부모 트리로 보아야 한다.
- Docker 컨테이너 이미지로 올라간 게임 서버가 있으면 해당 게임 서버 옆에서 콘솔을 볼 수 있어야 한다.
- 서버 선택 후 상세 화면에 들어갔을 때 Agent 상태 확인은 자동으로 한 번 실행되어야 자연스러운 UX가 된다.
- 테스트 서버에 설치된 18080 Agent 포트가 보안 문제가 될 수 있으므로 조치가 필요하다.
- Agent 보안 조치안을 문서화한 뒤 승인 여부를 결정한다.
- 사용자는 토큰 발급/적용을 프로그램 내부에서 자동 처리하길 원한다.
- 사용자는 서버 삭제 시 Agent 서비스 종료/삭제, 18080 방화벽 닫기 후 SSH 등록 정보를 삭제하는 순서를 원한다.
- 사용자는 내부망 서버에 점프 서버를 통해 접근해야 하는 환경을 서버 추가 옵션으로 지원하길 원한다.
- 사용자는 서버 등록 조건을 Agent 확인 성공으로 제한하고, 서버 등록 전 SSH 확인과 Agent 설치/포트 설정을 강제하길 원한다.
- 사용자는 버튼 클릭 가능 여부가 마우스 커서와 시각 반응으로 보이고, 요청 처리 중 로딩 상태가 표시되길 원한다.
- 사용자는 점프 서버 경유 환경에서 Agent 포트 개방도 실제 설치 서버까지 넘어가 적용되길 원한다.
- 사용자는 등록된 서버 상세 화면에서 `Agent 설치` 버튼은 불필요하고 `Agent 업데이트`만 있으면 된다고 판단한다.
- 사용자는 점프 서버가 포트 중계 역할을 할 때 `iptables DNAT`를 사용할 수 있다는 점을 확인했고, DNAT는 기본 Agent 관리 경로가 아니라 게임 서버 포트 공개 또는 명시 승인된 직접 공개 예외로 계획에 반영한다.
- 사용자는 Agent `18080` 포트 중계는 proxy TCP 방식이 좋겠다고 제안했고, Agent 관리 포트는 DNAT보다 TCP proxy 직접 공개 예외를 우선 검토하는 방향으로 계획에 반영한다.
- Agent `18080` TCP proxy 구현은 HAProxy를 우선 후보로 조정한다. HAProxy는 Agent 관리 포트와 게임 서버 TCP 포트 중계를 같은 방식으로 관리할 수 있고, 서버별 SSH tunnel systemd 서비스와 HAProxy frontend/backend 설정으로 구성한다.
- 사용자는 UDP/TCP 경유는 무조건 HAProxy를 사용하고, HAProxy는 외부망에 접근 가능한 노트북 또는 점프 노드에 설치해 내부망 서버로 연결하는 구조를 원한다. 구현 문서는 TCP/UDP 경유를 HAProxy 설정으로 통일하고, UDP는 HAProxy UDP 지원 확인 후에만 등록하는 방향으로 수정한다.
