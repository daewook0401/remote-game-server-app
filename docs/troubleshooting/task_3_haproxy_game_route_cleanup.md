# HAProxy 게임 포트 라우트 정리

## 증상

- Minecraft 컨테이너를 `25580/tcp`로 생성한 뒤 종료/삭제해도 `/etc/haproxy/haproxy.cfg`에 `bind 0.0.0.0:25580` 게임 라우트가 남을 수 있었다.
- 이 상태는 컨테이너가 없어도 경유 서버의 HAProxy가 해당 포트를 계속 리슨하거나, 다음 생성 시 설정 충돌/혼동을 만들 수 있다.

## 원인

- 컨테이너 삭제 흐름에서 내부망 서버와 경유 서버의 방화벽 포트 정리는 수행했지만, HAProxy 설정 파일의 게임 route 제거는 별도 수행하지 않았다.
- 서버 삭제는 HAProxy 관리 블록 전체를 지우는 흐름이 있었지만, 컨테이너 삭제는 서버의 Agent route `18080/tcp`를 유지해야 하므로 전체 블록 삭제를 그대로 사용할 수 없었다.

## 조치

- 컨테이너 삭제 후 외부망 HAProxy 경유 서버라면 해당 게임 route id만 제거하도록 분리했다.
- 예: `25580/tcp` 삭제 시 `game-25580-tcp` route만 제거하고 Agent route는 유지한다.
- 기존에 서버 이름 기반 ID 변경으로 쌓인 예전 HAProxy 관리 블록도 같은 내부 서버 IP와 같은 route id가 있으면 함께 정리될 수 있도록 했다.
- 삭제 모달에서 방화벽을 유지하더라도 HAProxy 경유 서버의 SSH password가 필요한 경우 입력받도록 변경했다.

## 확인 방법

1. 외부망 HAProxy 경유 서버에서 Minecraft 컨테이너를 생성한다.
2. `/etc/haproxy/haproxy.cfg`에 `bind 0.0.0.0:<게임포트>`가 생성되는지 확인한다.
3. 앱에서 컨테이너를 삭제한다.
4. 삭제 후 `haproxy.cfg`에서 해당 게임 포트 route가 사라지고, Agent `18080/tcp` route는 유지되는지 확인한다.
